# Story 6.1: PWA Configuration & iPhone Home Screen Install

Status: done

## Story

As a Wolf Cup scorer or spectator,
I want to install the app to my iPhone home screen via Safari,
so that I can launch Wolf Cup in full-screen mode without browser chrome, just like a native app.

## Acceptance Criteria

### Service Worker & Offline Capability

1. `pnpm --filter @wolf-cup/web build` produces `dist/sw.js` and at least one `dist/workbox-*.js` file — confirming the service worker was generated by Workbox.

2. The app shell (HTML, JS bundles, CSS, PNG icons) is precached by the service worker on first visit, matching all files in `dist/**/*.{js,css,html,ico,png,svg,woff,woff2}` (NFR17: app shell loads from cache instantly on repeat visits).

3. API routes `/api/leaderboard*`, `/api/standings*`, `/api/stats*`, `/api/rounds*` are cached with a **NetworkFirst** strategy: fresh network response served when available; after a 3-second network timeout (or when offline), the last cached response is served instead (NFR20: read-only mode survives temporary server outage).

4. The service worker takes control of all open tabs immediately after install and on update — no manual page reload required (`clientsClaim: true`, `skipWaiting: true` in the Workbox config block).

### Manifest & iPhone Install

5. The production `dist/manifest.webmanifest` contains:
   - `name: "Wolf Cup"`
   - `short_name: "Wolf Cup"`
   - `display: "standalone"` (full-screen, no browser chrome)
   - `theme_color: "#1a1a1a"`
   - `background_color: "#ffffff"`
   - `start_url: "/"`
   - Icons: 192×192 and 512×512 PNG entries

6. `apps/web/index.html` already contains the required Apple-specific meta tags:
   - `<meta name="apple-mobile-web-app-capable" content="yes" />` ✓
   - `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />` ✓
   - `<meta name="apple-mobile-web-app-title" content="Wolf Cup" />` ✓
   - `<link rel="apple-touch-icon" href="/icon-192.png" />` ✓
   - `<link rel="manifest" href="/manifest.webmanifest" />` ✓

   **No changes needed to `index.html`** — these are already present from Story 3.1.

### Offline Indicator

7. The app header displays a compact "Offline" badge (red indicator + "Offline" text) when `navigator.onLine === false`. The badge disappears immediately when connectivity is restored. The badge uses `position: absolute` or inline within the header — does not push layout content.

8. Online/offline detection uses `window.addEventListener('online', ...)` / `window.addEventListener('offline', ...)` event listeners with proper cleanup on unmount.

### Code Quality

9. `pnpm --filter @wolf-cup/api typecheck`, `pnpm --filter @wolf-cup/web typecheck`, and `pnpm lint` all pass with no errors.

### Task 1: Complete Workbox Config

Add the `workbox` block to the existing VitePWA call in `vite.config.ts`:

```typescript
VitePWA({
  registerType: 'autoUpdate',
  devOptions: {
    enabled: true,
    type: 'module',
  },
  manifest: {
    // ... existing manifest unchanged ...
  },
  workbox: {
    clientsClaim: true,
    skipWaiting: true,
    globPatterns: ['**/*.{js,css,html,ico,png,svg,woff,woff2}'],
    runtimeCaching: [
      {
        urlPattern: /\/api\//i,
        handler: 'NetworkFirst',
        options: {
          cacheName: 'wolf-api-cache',
          networkTimeoutSeconds: 3,
          expiration: {
            maxEntries: 50,
            maxAgeSeconds: 300, // 5 minutes
          },
          cacheableResponse: {
            statuses: [0, 200],
          },
        },
      },
    ],
  },
}),
```

**Key placement rules (vite-plugin-pwa 1.x):**
- `registerType` → top-level VitePWA option
- `devOptions` → top-level VitePWA option
- `manifest` → top-level VitePWA option
- `clientsClaim`, `skipWaiting`, `globPatterns`, `runtimeCaching` → inside `workbox` block
- `handler` must be a string (`'NetworkFirst'`), not a class import

**Why NetworkFirst for API routes:**
- Scorers always get fresh leaderboard/standings when online
- When offline or server unreachable (NFR20), last cached response served immediately after 3s timeout
- Covers `/api/leaderboard`, `/api/standings`, `/api/stats`, `/api/rounds` with one pattern

**Why `globPatterns` explicit:**
Default auto-discovery works, but explicit patterns ensure fonts and icons are included. The `dist/` directory is the default `globDirectory` in generateSW mode — no need to specify it.

Updated header with offline badge:
```tsx
import { useState, useEffect } from 'react';

// Inside the component (or extract as a small hook):
const [isOnline, setIsOnline] = useState(() => navigator.onLine);

useEffect(() => {
  const handleOnline = () => setIsOnline(true);
  const handleOffline = () => setIsOnline(false);
  window.addEventListener('online', handleOnline);
  window.addEventListener('offline', handleOffline);
  return () => {
    window.removeEventListener('online', handleOnline);
    window.removeEventListener('offline', handleOffline);
  };
}, []);
```

**Note:** The `createRootRoute` component is an arrow function returning JSX. Move state/effect into an extracted component called `RootComponent` to use hooks properly:

```tsx
export const Route = createRootRoute({
  component: RootComponent,
});

function RootComponent() {
  const [isOnline, setIsOnline] = useState(() => navigator.onLine);

  useEffect(() => {
    const handleOnline = () => setIsOnline(true);
    const handleOffline = () => setIsOnline(false);
    window.addEventListener('online', handleOnline);
    window.addEventListener('offline', handleOffline);
    return () => {
      window.removeEventListener('online', handleOnline);
      window.removeEventListener('offline', handleOffline);
    };
  }, []);

  return (
    <div className="flex flex-col min-h-screen bg-background text-foreground">
      <header className="sticky top-0 z-50 bg-background border-b px-4 py-3">
        <div className="flex items-center justify-between">
          <h1 className="text-lg font-bold">🐺 Wolf Cup</h1>
          {!isOnline && (
            <span className="flex items-center gap-1 text-xs text-destructive font-medium">
              <span className="h-2 w-2 rounded-full bg-destructive inline-block" />
              Offline
            </span>
          )}
        </div>
      </header>

      <main className="flex-1 overflow-y-auto">
        <Outlet />
      </main>

      <nav className="sticky bottom-0 border-t bg-background grid grid-cols-4 divide-x">
        {/* ... all 4 nav links unchanged ... */}
      </nav>
    </div>
  );
}
```

**Important:** The current `__root.tsx` uses an anonymous arrow function as the `component`. Hooks cannot be used inside anonymous arrow functions passed directly to `createRootRoute`. Extract to a named function `RootComponent` as shown above.
